# Watermark Remover

本项目是一个本地运行的图片去水印、主体抠图和分辨率增强工具，包含前端工作台、FastAPI 后端、启动器和本地模型缓存目录。

## 一键启动

双击 `start.bat`。

启动器会检测本机 conda 环境，并提示选择运行环境。推荐输入 `0`：

- `0 / wmr310`：自动创建或使用项目专用 Python 3.10.11 环境，兼容 Real-ESRGAN、SwinIR、diffusers、BEN2 等模型依赖。
- 已有 CUDA PyTorch 环境：可以启动基础功能，但可能无法完整体验增强放大、生成式修复、BEN2 等模型。
- 无 CUDA 环境：启动器会尝试安装固定 CUDA PyTorch。

推荐环境会锁定核心依赖，避免 pip 反复替换 torch：

```text
Python 3.10.11
torch==2.7.1+cu118
torchvision==0.22.1+cu118
numpy==2.2.2
```

首次安装依赖和首次下载模型可能较慢。启动器和前端都会显示进度；如果中途取消，下次重新运行通常可以继续使用 conda/pip/Hugging Face 缓存。

启动后浏览器会打开：

```text
http://127.0.0.1:8000
```

## 前端功能

- 上传或拖入 JPG / PNG / WebP 图片。
- 涂抹水印区域，支持画笔大小、撤销、清空、涂抹/擦除。
- 原图和结果滑动对比。
- 复制原图、复制结果图。
- 把结果重新放回上传区继续处理，不需要先下载再拖入。
- 下载结果图片。
- 前端可直接停止服务，不必回终端按 `Ctrl+C`。
- 页面打开后会自动做轻量 CUDA warmup，减少第一次调用模型时的等待。

## 去水印模型

- OpenCV 快速修复：无需下载，速度最快，适合简单背景和小水印。
- LaMa 本地修复：适合自然纹理和中等水印，首次使用会下载 `models/lama/big-lama.pt`。
- SD 1.5 Inpaint 生成式修复：适合大块水印和需要重绘的区域，首次使用会下载到 `models/diffusers/`。

生成式修复提示词可以留空，也可以在前端选择示例提示词。主体需求是“尽量去掉水印并保持原图自然”，提示词不一定越复杂越好。

## 主体抠图

前端提供两个抠图模型：

- 快速 u2net：速度快、兼容性好，默认选项，模型保存到 `models/rembg/`。
- 高质量 BEN2：适合头发、复杂边缘、商品图和高分辨率图片，模型保存到 `models/ben2/`。

BEN2 首次使用会下载模型；下载或推理失败时不会影响 u2net 的使用。

## 分辨率增强

增强倍率支持 `0-8x` 自由选择。当前保留四个增强模型：

- R-ESRGAN 4x+：通用照片、自然图像、老图修复。
- R-ESRGAN 4x+ Anime6B：动漫、插画、线稿、二次元图。
- 4x-UltraSharp：JPEG 压缩图、AI 图、需要锐化的图。
- SwinIR-L 4x：真实退化、复杂纹理、结构一致性。

Real-ESRGAN 系列按官方 `realesrgan.RealESRGANer + basicsr RRDBNet` 调用。默认优先使用快速整图推理；只有显存不足时才自动降级到分块推理。带透明通道的图片不会再让模型额外推理 alpha 通道，alpha 会单独缩放合并。

UltraSharp 默认优先整图推理，显存不足时再降级为大块分块。SwinIR-L 默认优先使用较大的 tile，显存不足时再逐步降级。

## 模型和缓存目录

依赖、模型和缓存会尽量保存在项目目录内：

```text
models/
models/rembg/
models/ben2/
models/lama/
models/diffusers/
models/realesrgan/
models/.cache/
.cache/
```

常见大模型首次下载耗时较长，前端会显示下载或处理进度。网络中断后再次启动或再次使用模型时，会尽量复用已有缓存。

## 退出服务

推荐直接在前端点击 `停止服务`。按钮需要二次确认，避免误触；确认后会立即向后端发送关闭指令并更新前端状态。

也可以在启动终端按 `Ctrl+C` 停止服务。如果 Windows 显示：

```text
Terminate batch job (Y/N)?
```

输入 `Y` 并按回车即可退出。

## 常见说明

- 如果 NumPy 版本不匹配，启动器会提示安装 `numpy==2.2.2` 或切换到依赖正确的 conda 环境。
- 如果没有 CUDA，OpenCV 等基础功能仍可用，但 AI 模型速度和可用性会受影响。
- 推荐使用 `wmr310` 环境获得完整体验。
- 下载模型时保持网络稳定；如果 Hugging Face 下载较慢，可以配置代理后重新启动。
